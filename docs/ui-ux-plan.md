# UI/UX 개선안 — 체계 정리 (2026-07-10)

방향은 `docs/mockups/direction-a.html`(교정지)로 확정됐다. 원칙 한 줄:
**도구의 껍데기가 산출물과 같은 언어를 쓴다.** 이 도구가 만드는 슬라이드는
흰 종이에 먹색 본문, 교과서 파랑 마커다. 껍데기도 그 잉크로 만든다.

각 항목은 [상태] 근거 → 내용 → 효과 순으로 적는다.

---

## 1단계 — 기반 (이번에 구현)

### 1-1. 교정지 팔레트 [완료]
근거: 기존 #6B3F26/#EEDDD0은 AI 생성 디자인의 대표 기본값이며, 산출물의
시각 언어(파랑 마커 #294C67·흰 슬라이드)와 무관했다.
내용: 종이 #FBFAF7 · 먹 #16202B · 연필 #5B6470 · 교과서 파랑 #294C67
(hover #21405A) · 괘선 #E4E1DA · 선택 틴트 #E8EFF5. `src/` 내 구 팔레트 0건.
마커·산출물 색은 불변(콘텐츠이므로).

### 1-2. 접근성 대비 [완료]
근거: 실측 — `text-[#6B3F26]/50` 2.52:1(17곳), `/40` 2.04:1, `/60` 3.15:1,
"키 설정" 버튼 2.15:1. WCAG AA 기준 4.5:1.
내용: 불투명도 기반 회색을 실색 단계로 교체(연필 #5B6470 = 5.7:1,
힌트 #6E7683 = 4.8:1), 키 버튼 amber-800(7.1:1).

### 1-3. 동작 중심 카피 [완료]
근거: 컨트롤은 눌렀을 때 일어나는 일을 말해야 한다.
내용: "PPT 생성"→"PPT 내보내기"(만드는 게 아니라 꺼내는 것),
"주석 추출 + 분할"→"PDF에서 주석 가져오기"(한 버튼=한 동작),
키 배너에 비용·소요 시간 명시("무료이고 1분"), 오류문은 다음 행동을 안내.

### 1-4. 슬라이드 설정 패널 [구현 중]
근거: **설정을 편집하는 UI가 없었다.** pptSettings는 상태로만 존재했고
AnnotationEditor는 그마저 무시하고 장르 기본값을 재파생(90행 버그).
내용: 사이드바에 접이식 패널 — 글꼴(실측 캘리브레이션된 6종),
글자 크기 24–44, 줄간격 1.2–2.2. 즉시 반영·자동 저장.
옛한글 필요 시 한컴산뜻돋움 안내. 90행 버그 수정 포함.
효과: 마커 기하가 전 구간 실측 검증됐으므로(±3px) 이제 설정을 열어도 안전.
**설정 개방은 기하 수정의 후속 조치다 — 순서가 반대였으면 사고였다.**

### 1-5. 리허설 모드 [구현 예정]
근거: 에디터에 미리보기가 없어 PowerPoint를 열기 전까지 결과를 모른다.
목업의 시그니처 — "투사면을 편집한다".
내용: 주석 화면에서 리허설 시작 → 흰 슬라이드 모사 화면에서 클릭마다
주석이 order 순으로 등장(교실에서 벌어질 시퀀스 그대로). Esc 종료.
`prefers-reduced-motion` 시 전체 정지 표시.

## 2단계 — 다음 (설계만, 별도 회차)

### 2-1. 여백 주석(marginalia) 레이아웃 [설계 완료, 미구현]
목업의 본체. 주석이 오른쪽 목록이 아니라 자기가 가리키는 행 옆 여백에
지시선으로 매달린다. 관계가 조회가 아니라 공간이 된다.
미구현 사유: AnnotationEditor(1,036줄) 재구조화가 선행돼야 한다 —
5개 상호배타 인터랙션 모드가 독립 불리언으로 얽혀 있어(아키텍처 렌즈 지적)
야간 무인 작업으로는 회귀 위험이 크다.
선행 작업: AnnotationEditor를 모드 상태기계 + 프레젠테이션 컴포넌트로 분리.

### 2-2. AI 출처 배지 + 내보내기 전 확인 [미착수]
근거: AI가 뽑은 주석과 교사가 쓴 주석이 화면에서 구분되지 않는다.
잘못 추출된 주석이 교실 스크린에 뜨는 순간이 신뢰 위기의 시작.
내용: Annotation에 source: "ai" | "manual" 필드(마이그레이션: 기존은 manual),
여백/목록에 배지, 내보내기 직전 "AI 주석 N개 확인" 1회 체크.

### 2-3. 키보드 전용 주석 작성 [미착수]
근거: 핵심 상호작용이 onMouseUp 전용 — 키보드만으로 주석을 하나도 못 만든다
(접근성 렌즈, critical). 내용: 텍스트 커서 이동+Shift 선택 → 단축키로 마커 부여.
2-1의 재구조화와 함께 설계하는 것이 경제적.

### 2-4. 온보딩 재배치 [미착수]
근거: 키가 필요한 Mode C가 "(추천)" 기본이라 첫 방문자가 곧장 키 벽에 부딪힌다.
Mode B(키 불필요)가 이 도구의 확실한 가치(자동 조판+클릭 애니메이션)를
키 없이 증명할 수 있다.
내용: 첫 화면에서 "지금 바로 시작(키 불필요)" 경로를 전면에, PDF 가져오기는
"이미 써 본 사람"의 다음 단계로. 홈 화면 워딩·순서 재설계와 함께.

### 2-5. 홈 화면 팔레트 정합 [부분]
1-1에서 에디터는 완전 이행됐으나 홈의 보조 웜톤(#8B6040 계열)은 별도
기획 문서와 겹쳐 보류. 홈 재설계(2-4)와 함께 일괄 처리.

## 하지 않기로 한 것

- **다크 모드**: 산출물이 흰 슬라이드다. 어두운 에디터는 결과물에 대해
  거짓말을 한다(방향 B 기각 사유와 동일).
- **애니메이션 장식**: 모션은 리허설 하나에만 쓴다. 흩어진 효과는 소음.
- **numbered step 마커(01/02/03) 남용**: 순서가 정보일 때(클릭 순서)만 번호.
